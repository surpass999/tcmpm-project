# 综合型项目过程指标配置方案

## 概述

本文档详细说明综合型项目（project_type=1）在各个过程类型中的指标配置方案，用于测试项目审批流程。

## 指标逻辑校验规则

系统配置了以下联合验证规则，用于确保数据一致性：

| 规则名称 | 触发时机 | 验证逻辑 |
|---------|---------|---------|
| 资金到账金额校验 | 提交时 | 中央转移支付到账 ≤ 项目总投资 |
| 资金使用不超过到账金额 | 提交时 | 中央资金使用 ≤ 到账金额 |
| 累计投资不超过总投资 | 提交时 | 累计完成投资 ≤ 总投资 |
| 已完成系统数不超过计划数 | 提交时 | 已完成系统数 ≤ 计划数 |
| 已完成模块数不超过计划数 | 提交时 | 已完成模块数 ≤ 计划数 |
| 等保系统数求和校验 | 提交时 | 三级 + 二级 = 等保备案总数 |
| 数据集完成数不超过计划数 | 提交时 | 已完成数据集 ≤ 计划数 |
| 资金执行率自动计算 | 填报时 | 自动计算 |
| 中央资金执行率自动计算 | 填报时 | 自动计算 |
| 功能覆盖率自动计算 | 填报时 | 自动计算 |

## 测试流程建议

1. **初始化数据**：执行 `init_indicator_data_for_testing.sql` 创建指标基础数据
2. **配置过程指标**：执行 `init_process_indicator_config.sql` 配置各过程指标
3. **创建综合型项目**：在系统中创建综合型项目
4. **按流程测试**：
   - 建设过程 → 半年报 → 年度总结 → 中期评估 → 整改记录 → 验收申请
