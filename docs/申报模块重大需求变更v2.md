# 申报模块重大需求变更 - 精简实施方案

## 一、业务理解确认

### 业务背景

中医药传承项目管理系统，用于管理国家中医药传承项目的进度填报和成果管理。

### 核心业务（4个独立模块）

| 模块 | 发起方 | 流程 | BPM |
|------|--------|------|-----|
| **进度填报** | 国家局发布窗口 → 医院填报 → 省局审批 → 推送国家局 | 定期上报数据 | ⏳ 待配置 |
| **成果遴选** | 医院随时提交成果 → 省局审批 → 推送国家局 | 独立申报，依托医院 | ✅ 沿用旧key |
| **政策通知** | 国家局发布 | 下发通知 | 不需要 |
| **活动交流** | 国家局发布 | 发布培训/活动 | 不需要 |

### 医院定位

- **医院 = 填报主体/申报主体**
- 一个医院 ≈ 一个独立项目（不需要项目概念）
- 医院有自己的属性：分型、等级、统一社会信用代码等

### 删除的模块

| 模块 | 删除原因 |
|------|---------|
| ~~项目管理~~ (project) | 不需要项目概念，医院即主体 |
| ~~备案~~ (filing) | 已确认不需要 |
| ~~专家管理~~ (expert) | 成果遴选不需要专家评审 |
| ~~评审流程~~ (review) | 省局管理员直审即可 |

### 精简后架构

```

│
```

### 本次删除

| # | 需求 | 处理方式 |

| 1 | 项目管理 | 删除 project 相关代码 |

| 3 | 专家管理 | 删除 expert 相关代码 |
| 4 | 评审流程 | 删除 review 相关代码（成果遴选改为省局直审） |

### 保留功能

| 模块 | 说明 |
|------|------|
| 成果遴选 | 沿用旧 BPM key，已配置好 |
| 政策通知 | 保持现状 |
| 活动交流 | 保持现状 |
| 医院管理 | 保持现状 |

### 关键细节确认

- **填报窗口**: 国家局设置开放时间段，窗口期内医院可随时填报
- **填报次数**: 无限制（只要窗口开放，医院可多次填报/修改，驳回后重新提交即可）
- **填报内容**: 每次填报全量指标
- **上期值展示**: 显示上期值 + 变化量（用颜色区分增长/下降）
- **批量上报范围**: 同省跨医院多选批量上报
- **国家局处理**: 上报后只需查看/导出，无需审核
- **不需要表单附表**: 每次填报只有指标数据，无富文本、无附件
- **标签维度**: 区域（华东区等）、等级（三级甲等等）、特征（综合型等）

## 三、数据库设计

### 3.0 医院管理表（system_hospital）

> 医院信息表，用于存储医院的基本信息、分型、等级、归属地等

```sql
-- =============================================
-- 医院信息表
-- =============================================
CREATE TABLE system_hospital (
    id                              BIGINT NOT NULL AUTO_INCREMENT COMMENT '主键ID',
    dept_id                         BIGINT COMMENT '关联部门ID（对应system_dept.id）',

    -- 医院基本信息
    hospital_code                   VARCHAR(50) COMMENT '医院编码（统一社会信用代码作为唯一标识）',
    hospital_name                   VARCHAR(200) NOT NULL COMMENT '医院全称',
    short_name                      VARCHAR(100) COMMENT '医院简称',

    -- 证照信息
    unified_social_credit_code     VARCHAR(50) COMMENT '统一社会信用代码（18位，唯一标识）',
    medical_license_no              VARCHAR(100) COMMENT '医疗机构执业许可证登记号',

    -- 分类信息
    hospital_type                   VARCHAR(50) COMMENT '医院分型：综合型、名老中医、专科等',
    hospital_level                  VARCHAR(50) COMMENT '医院等级：三级甲等、三级乙等、二级甲等、二级乙等、一级等',
    hospital_category               VARCHAR(50) COMMENT '医院类别：综合医院、中医医院、中西医结合医院等',

    -- 归属地信息
    province_code                   VARCHAR(20) COMMENT '所属省份编码',
    province_name                   VARCHAR(50) COMMENT '所属省份名称',
    city_code                       VARCHAR(20) COMMENT '所属城市编码',
    city_name                       VARCHAR(50) COMMENT '所属城市名称',
    district_code                   VARCHAR(20) COMMENT '所属区县编码',
    district_name                   VARCHAR(50) COMMENT '所属区县名称',

    -- 联系方式
    contact_person                  VARCHAR(50) COMMENT '联系人',
    contact_phone                   VARCHAR(20) COMMENT '联系电话',

    -- 规模信息
    bed_count                       INT COMMENT '编制床位数',
    employee_count                  INT COMMENT '在职职工人数',

    -- 状态
    status                          TINYINT NOT NULL DEFAULT 1 COMMENT '状态：0-停用 1-启用',

    -- 审计字段
    creator                         VARCHAR(64) DEFAULT '',
    create_time                     DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updater                         VARCHAR(64) DEFAULT '',
    update_time                     DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted                         BIT(1) NOT NULL DEFAULT b'0',

    -- 索引
    UNIQUE KEY uk_dept_id (dept_id),
    UNIQUE KEY uk_credit_code (unified_social_credit_code),
    INDEX idx_province (province_code),
    INDEX idx_level (hospital_level),
    INDEX idx_status (status),

    PRIMARY KEY (id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='医院信息表';
```

### 3.4 上期值查询逻辑

```sql
-- SQL：获取某医院某指标的上期值
SELECT
    div.*
FROM declare_indicator_value div
INNER JOIN declare_progress_report dpr
    ON div.business_id = dpr.id
    AND div.business_type = 10  -- 进度填报类型
WHERE
    div.hospital_id = #{hospitalId}
    AND div.report_year = #{reportYear}
    AND dpr.report_batch = (
        SELECT MAX(dpr2.report_batch)
        FROM declare_progress_report dpr2
        WHERE dpr2.hospital_id = #{hospitalId}
          AND dpr2.report_year = #{reportYear}
          AND dpr2.report_batch < #{currentBatch}
          AND dpr2.province_status = 2  -- 省级通过
          AND dpr2.deleted = FALSE
    )
    AND dpr.province_status = 2
    AND dpr.deleted = FALSE;
```

```java
public interface HospitalTagQueryService {
```

### 6.2 SQL 查询示例

```sql
-- 查询华东区 + 三级甲等 的所有医院
SELECT DISTINCT h.*
FROM declare_hospital h
INNER JOIN declare_hospital_tag_relation r1 ON h.social_credit_code = r1.hospital_code
INNER JOIN declare_hospital_tag t1 ON r1.tag_id = t1.id AND t1.tag_code = 'REGION_EAST'
INNER JOIN declare_hospital_tag_relation r2 ON h.social_credit_code = r2.hospital_code
INNER JOIN declare_hospital_tag t2 ON r2.tag_id = t2.id AND t2.tag_code = 'LEVEL_3A';

-- 按区域统计医院数量
SELECT t.tag_name AS region, COUNT(DISTINCT r.hospital_code) AS hospital_count
FROM declare_hospital_tag t
LEFT JOIN declare_hospital_tag_relation r ON t.id = r.tag_id
WHERE t.tag_category = 'region'
GROUP BY t.id, t.tag_name
ORDER BY hospital_count DESC;

-- 跨维度统计：各区域内各等级医院数量
SELECT
    r1.tag_name AS region,
    r2.tag_name AS level,
    COUNT(DISTINCT tr.hospital_code) AS count
FROM declare_hospital_tag_relation tr
INNER JOIN declare_hospital_tag r1 ON tr.tag_id = r1.id AND r1.tag_category = 'region'
INNER JOIN declare_hospital_tag_relation tr2 ON tr.hospital_code = tr2.hospital_code
INNER JOIN declare_hospital_tag r2 ON tr2.tag_id = r2.id AND r2.tag_category = 'level'
GROUP BY r1.id, r1.tag_name, r2.id, r2.tag_name;
```
