## 1. 普通数值比较

### 语法

```

指标A >= 指标B
指标A + 指标B > 指标C

```

直接写指标代码（可带中括号）配合比较运算符，支持加减乘除公式。

### 示例

| 规则 | 说明 |
|------|------|
| `201>=20101` | 指标201的数值应大于等于指标20101 |
| `802>=80201+80202` | 指标802应大于等于指标80201+80202之和 |
| `301>=201*0.5` | 指标301应大于等于指标201的50% |

## 2. 条件验证 IF

### 语法

```

IF([条件指标] 条件 操作值, [验证指标] 验证操作 验证值, TRUE)

```

当条件满足时，要求验证指标满足指定条件。

### 条件操作符

| 操作符 | 说明 |
|--------|------|
| `==` | 等于 |
| `!=` | 不等于 |
| `>` | 大于 |
| `>=` | 大于等于 |
| `<` | 小于 |
| `<=` | 小于等于 |
| `IN(v1,v2,...)` | 在列表中 |
| `NOT_IN(v1,v2,...)` | 不在列表中 |

### IN / NOT_IN 详解

- **IN(v1,v2,...)**：条件指标的值在指定列表中时触发
- **NOT_IN(v1,v2,...)**：条件指标的值不在指定列表中时触发

值列表支持多个值，用英文逗号分隔，值可以是任意数字或指标编码。

### 示例

| 规则 | 说明 |
|------|------|
| `IF([601]==1, [602]>=0, TRUE)` | 当601=1时，602应>=0 |
| `IF([601]!=1, [602]>=0, TRUE)` | 当601≠1时，602应>=0 |
| `IF([601]>0, [602]>=0, TRUE)` | 当601>0时，602应>=0 |
| `IF([601]>=0, [602]<=100, TRUE)` | 当601>=0时，602应<=100 |
| `IF([601]<10, [602]>=0, TRUE)` | 当601<10时，602应>=0 |
| `IF([601]<=5, [602]>=0, TRUE)` | 当601<=5时，602应>=0 |
| `IF([601]IN(1,2,3), [602]>=0, TRUE)` | 当601在列表{1,2,3}中时，602应>=0 |
| `IF([601]NOT_IN(99,100), [602]>=0, TRUE)` | 当601不在列表{99,100}中时，602应>=0 |
| `IF([601]==2, [602_01]+[602_02]>=10, TRUE)` | 当601=2时，容器字段602_01+602_02之和>=10 |

> 容器内字段引用格式：`[容器代码_字段代码]`，如 `[602_01]`，将自动展开为完整字段key。

## 3. 容器级联互斥约束 CC

### 语法

```

CC([字段1,字段2,字段3,...], 触发条件, 约束条件)

```

语义：完整扫描字段列表，找到**第一个**满足触发条件的字段，约束其**之后所有字段**必须满足（或不能等于）约束条件。

### 字段代码写法

在容器指标中，字段代码为子字段的**最后2位编码**。例如字段代码为 `01`，则写 `01`。

```

CC([01,02,03,04,05,06], FIRST_EQ(2), REST_NE(1))

```

### 触发运算符（triggerOp）

| 运算符 | 说明 |
|--------|------|
| `FIRST_EQ(val)` | 从前往后扫描，找到第一个值**等于** val 的字段 |
| `LAST_EQ(val)` | 从后往前扫描，找到最后一个值**等于** val 的字段 |
| `FIRST_GT(val)` | 从前往后扫描，找到第一个值**大于** val 的字段 |
| `FIRST_GTE(val)` | 从前往后扫描，找到第一个值**大于等于** val 的字段 |
| `FIRST_LT(val)` | 从往前扫描，找到第一个值**小于** val 的字段 |
| `FIRST_LTE(val)` | 从前往后扫描，找到第一个值**小于等于** val 的字段 |
| `FIRST_IN(v1,v2,...)` | 从前往后扫描，找到第一个值**在列表中**的字段 |
| `LAST_IN(v1,v2,...)` | 从后往前扫描，找到最后一个值**在列表中**的字段 |

### 约束运算符（forbidOp）

| 运算符 | 说明 |
|--------|------|
| `REST_EQ(val)` | 触发字段之后的字段**必须等于** val |
| `REST_NE(val)` | 触发字段之后的字段**不能等于** val |

### 场景一：单选题级联（选A后，B/C/D不能选A）

场景：字段01-04为单选题，选了某个值之后，后面的字段不能再选同一个值。

```

CC([01,02,03,04], FIRST_EQ(2), REST_NE(2))

```

含义：找到第一个选了值2的字段，之后的字段不能再选值2。

| 01 | 02 | 03 | 04 | 结果 |
|----|----|----|----|------|
| 2 | 1 | 1 | 1 | 通过 |
| 1 | 2 | 1 | 2 | 通过（02触发，03不能选2，但04没触发所以可选2） |
| 1 | 2 | 2 | 1 | 报错（03选了2，违反"02之后不能选2"的约束） |

### 场景二：多选数量上限

场景：字段01-06为多选（checkbox），只有第一个选了"2（有）"的字段，其后所有字段才允许选"1（无）"。

```

CC([01,02,03,04,05,06], FIRST_EQ(2), REST_NE(1))

```

含义：找到第一个选了值2的字段，之后的字段不能选值1。

| 01 | 02 | 03 | 04 | 结果 |
|----|----|----|----|------|
| 1 | 1 | 1 | 1 | 通过（无触发，所有字段可选1） |
| 2 | 1 | 1 | 1 | 通过（01触发，02-06不能选1，符合） |
| 2 | 1 | 2 | 1 | 报错（01触发，03选了2没问题，但04选了1违反约束） |
| 2 | 1 | 2 | 2 | 通过 |

### 场景三：条件触发（从后往前找）

场景：字段01-06，从后往前扫描，找到最后一个选了2的字段，其后所有字段（之后没有了，即触发字段本身）不能选1。

```

CC([01,02,03,04,05,06], LAST_EQ(2), REST_NE(1))

```

含义：从后往前找最后一个选了值2的字段，之后的字段（仅该字段之后）不能选值1。
因为是从后往前扫，"之后"范围更小，通常用于反向约束。

### 场景四：IN 列表触发

场景：字段01-06，找到第一个选了列表中任意值的字段，之后的字段不能选1。

```

CC([01,02,03,04,05,06], FIRST_IN(2,3,4), REST_NE(1))

```

含义：找到第一个选了2、3或4中任意一个值的字段，之后的字段不能选1。

## 4. 文件格式多样性验证 FORMATS

### 语法

```

FORMATS(["类别1","类别2"])

```

验证该指标已上传的文件必须覆盖指定的所有格式类别。

### 简单用法（无 typeGroups）

直接指定扩展名为类别名：

```

FORMATS(["doc","pdf"])

```

含义：必须同时上传 `.doc`（或 `.docx`）和 `.pdf` 文件。

验证行为：

| 上传文件 | 覆盖类别 | 结果 |
|---------|---------|------|
| `report.docx` + `report.pdf` | doc, pdf | 通过 |
| `report.pdf` | pdf | 不通过，缺少 doc |
| `report.docx` | doc | 不通过，缺少 pdf |

### 进阶用法（带 typeGroups）

将多个扩展名归并为同一个格式类别：

```

FORMATS(["word","pdf"], {word:["doc","docx"], pdf:["pdf"]})

```

第一个参数 `["word","pdf"]`：必须包含的格式类别列表
第二个参数 `{...}`：扩展名到格式类别的映射（可省略，用默认值）

含义：必须同时包含"word"类别和"pdf"类别的文件。`doc` 和 `docx` 都归并为 `word`。

### 常见 typeGroups 映射

```json
{
  "word":  ["doc", "docx"],
  "pdf":   ["pdf"],
  "excel": ["xls", "xlsx"],
  "image": ["jpg", "jpeg", "png", "gif", "bmp", "webp"],
  "ppt":   ["ppt", "pptx"],
  "zip":   ["zip", "rar", "7z"]
}
```

### 完整示例

```

FORMATS(["word","pdf","image"], {word:["doc","docx"], pdf:["pdf"], image:["jpg","png","gif"]})

```

含义：必须同时包含 word、pdf、image 三种格式的文件。
