# 验证规则

## 普通数值比较

### 指标间比较

```
## [201] >= [20101]
## [201] >= [20101] AND [202] >= [20202]
```

### 指标与固定值比较

```
## [901] >= 100
## [JS003] <= 100
```

### 支持公式运算（加减乘除）

```
## [802] >= [80201] + [80202]
## [301] >= [201] * 0.5
## [902] >= [90201] - [90202]
## [903] >= [90301] / [90302]
```

## 条件验证 IF

### 基本格式

```
## IF([条件指标] 条件 操作值, [验证指标] 验证操作 验证值, TRUE)
```

当条件满足时，要求验证指标满足指定条件。

### 条件操作符

| 操作符 | 说明 |
|--------|------|
| `==` | 等于 |
| `!=` | 不等于 |
| `>` | 大于 |
| `>=` | 大于等于 |
| `<` | 小于 |
| `<=` | 小于等于 |
| `IN(v1,v2,...)` | 在列表中 |
| `NOT_IN(v1,v2,...)` | 不在列表中 |

### 示例

```
## IF([703] > 0, [704] > 0, TRUE)
## IF([703] == 0,[704] == 0,TRUE)
## IF([703] != 0, [704] > 0, TRUE)
## IF([601] < 5, [602] >= 0, TRUE)
## IF([601] <= 10, [602] >= 0, TRUE)
## IF([601] IN(1,2,3), [602] >= 0, TRUE)
## IF([601] NOT_IN(99,100), [602] >= 0, TRUE)
```

### 动作侧支持 IN / NOT_IN

验证指标可以使用 IN / NOT_IN 操作符，适用于多选题联动场景：

```
## IF([801] IN(1,2,3,4,5,6,7,8,9), [802] NOT_IN(7), TRUE)
## IF([801] == 5, [802] IN(1,2), TRUE)
```

含义：当条件满足时，验证指标不能选择（或必须选择）指定值。

### 多动作

TRUE 之前可以放置多个逗号分隔的动作：

```
## IF([602_03] == 1, [602_04] == 1, [602_05] == 1, [602_06] == 1, TRUE)
```

含义：当 602_03 等于 1 时，602_04、602_05、602_06 都必须等于 1。

### 容器字段简写

容器内字段引用使用 `[容器代码_字段代码]` 格式，自动展开为完整字段 key：

```
## IF([601] == 2, [602_01] + [602_02] >= 10, TRUE)
```

## 容器级联互斥约束 CC

### 基本格式

```
## CC([字段1,字段2,字段3,...], 触发条件, 约束条件)
```

语义：完整扫描字段列表，找到**第一个**满足触发条件的字段，约束其**之后所有字段**必须满足（或不能等于）约束条件。

### 字段代码写法

字段代码为子字段的**最后2位编码**，或使用容器简写 `[容器_字段]`：

```
## CC([01,02,03,04,05,06], FIRST_EQ(2), REST_NE(1))
## CC([602_03, 602_04, 602_05, 602_06, 602_07], FIRST_EQ(2), REST_NE(1))
```

### 触发运算符（triggerOp）

| 运算符 | 说明 |
|--------|------|
| `FIRST_EQ(val)` | 从前往后扫描，找到第一个值**等于** val 的字段 |
| `LAST_EQ(val)` | 从后往前扫描，找到最后一个值**等于** val 的字段 |
| `FIRST_GT(val)` | 从前往后扫描，找到第一个值**大于** val 的字段 |
| `FIRST_GTE(val)` | 从前往后扫描，找到第一个值**大于等于** val 的字段 |
| `FIRST_LT(val)` | 从前往后扫描，找到第一个值**小于** val 的字段 |
| `FIRST_LTE(val)` | 从前往后扫描，找到第一个值**小于等于** val 的字段 |
| `FIRST_IN(v1,v2,...)` | 从前往后扫描，找到第一个值**在列表中**的字段 |
| `LAST_IN(v1,v2,...)` | 从后往前扫描，找到最后一个值**在列表中**的字段 |

### 约束运算符（forbidOp）

| 运算符 | 说明 |
|--------|------|
| `REST_EQ(val)` | 触发字段之后的字段**必须等于** val |
| `REST_NE(val)` | 触发字段之后的字段**不能等于** val |

## 文件格式多样性验证 FORMATS

### 基本格式

```
## FORMATS(["word","pdf"], {word:["doc","docx"], pdf:["pdf"]})
```

验证该指标已上传的文件必须覆盖指定的所有格式类别。

第一个参数必须包含的格式类别列表，第二个参数为扩展名到格式类别的映射。

### 简单用法

直接指定扩展名为类别名：

```
## FORMATS(["word","pdf"])
```

含义：必须同时上传 `.doc`（或 `.docx`）和 `.pdf` 文件。

### 常见 typeGroups 映射

| 类别 | 扩展名 |
|------|--------|
| word | doc, docx |
| pdf | pdf |
| excel | xls, xlsx |
| image | jpg, jpeg, png, gif, bmp, webp |
| ppt | ppt, pptx |
| zip | zip, rar, 7z |

## 非空校验 not_empty

支持非空校验操作符，用于检查指标是否有值：

```
## [601] not_empty
```

空值判定：undefined、null、空字符串、空数组、空对象均视为未填写。
