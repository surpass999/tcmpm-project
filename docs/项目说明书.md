# 中医药传承项目管理系统 - 项目说明书

## 一、项目概述

### 1.1 项目背景

中医药传承项目管理系统用于管理国家中医药传承项目的进度填报、成果管理等工作。系统支持多省份、多医院的统一管理，实现医院定期进度上报、省级审核、批量上报国家局的完整业务流程。

### 1.2 系统定位

- **医院** = 填报主体/申报主体
- 一个医院 ≈ 一个独立项目（不需要单独的项目概念）
- 医院有自己的属性：分型、等级、统一社会信用代码等

### 1.3 核心业务模块

| 模块 | 说明 | 状态 |
|------|------|------|
| 进度填报 | 国家局配置窗口 → 医院填报 → 省级审批 → 批量上报国家局 | ✅ 已实现 |
| 成果遴选 | 医院随时提交成果 → 省局审批 → 推送国家局 | ✅ 已实现 |
| 政策通知 | 国家局发布 | ✅ 已实现 |
| 活动交流 | 国家局发布培训/活动 | ✅ 已实现 |
| 医院管理 | 医院基本信息管理 | ✅ 已实现 |
| 医院标签 | 多维度标签查询统计 | ✅ 已实现 |

## 三、业务流程

### 3.1 进度填报流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                              进度填报完整流程                                  │
└─────────────────────────────────────────────────────────────────────────────┘

国家局配置时间窗口
        │
        ▼
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────────────────┐
│  填报窗口开放     │ ──▶ │    医院填报       │ ──▶ │  提交审核（触发BPM流程）      │
│  (report_window) │     │ (progress_report)│     │  hospitalProcessInstanceId   │
└─────────────────┘     └─────────────────┘     └─────────────────────────────┘
                                                              │
                                                              ▼
                        ┌─────────────────────────────┐     ┌─────────────────┐
                        │  省级审核通过                 │ ◀── │   省级审核       │
                        │  (nationalReportStatus=1)    │     │ (provinceAudit) │
                        └─────────────────────────────┘     └─────────────────┘
                                      │
                                      ▼
                        ┌─────────────────────────────┐
                        │    批量上报国家局             │
                        │  (national_report_record)    │
                        └─────────────────────────────┘
```

#### 国家局上报状态

### 3.2 BPM流程定义

进度填报使用两个独立的BPM流程：

| 流程Key | 流程名称 | 说明 |
|---------|----------|------|
| progressReportHospital | 医院内部审核流程 | 提交员填报 → 审核员审核 → 省级提交 |
| progressReportProvince | 省级审核流程 | 省级管理员审核 → 通过/驳回 |

### 3.3 成果遴选流程

```
医院提交成果
      │
      ▼
┌─────────────────┐
│   BPM审核流程     │ ──▶ 省局审批
│ (declare_achievement)
└─────────────────┘
      │
      ▼
省级审核通过 ──▶ 推荐上报国家局 ──▶ 入推广库
```

### 6.8 动态容器 JSON 数据规范

动态容器（value_type=12）的 `value_str` 字段存储 JSON 数组字符串，支持以下两种格式：

**格式一：通用结构（fieldCode/fieldValue）**

```json
[
  { "fieldCode": "doctorName", "fieldLabel": "姓名", "fieldValue": "张三" },
  { "fieldCode": "doctorTitle", "fieldLabel": "职称", "fieldValue": "主任医师" }
]
```

**格式二：结构化数据（具体业务字段）**

```json
[
  { "awardName": "科技进步一等奖", "awardLevel": "national", "awardYear": "2023", "awardDept": "骨科" }
]
```

**约束建议：**

1. fieldValue 推荐使用字符串类型，便于 MySQL JSON 函数精确查询
2. 单条 JSON 数组建议不超过 100 条记录
3. JSON 字符串总长度建议不超过 65535 字节（TEXT 字段限制）
4. 不推荐在 JSON 中存储大文件 URL（文件 URL 放在 value_str 根级别）
5. MySQL 5.7+ 可使用 `JSON_CONTAINS`、`JSON_EXTRACT`、`JSON_LENGTH` 等函数进行查询统计

**查询示例（MySQL 5.7）：**

```sql
-- 精确查询字段值包含"科技进步奖"的记录
SELECT * FROM declare_indicator_value
WHERE value_type = 12
  AND JSON_CONTAINS(value_str, '"科技进步奖"', '$.awardName');

-- 统计总条目数量
SELECT SUM(JSON_LENGTH(value_str))
FROM declare_indicator_value
WHERE business_type = 3 AND business_id = #{reportId} AND value_type = 12;
```
