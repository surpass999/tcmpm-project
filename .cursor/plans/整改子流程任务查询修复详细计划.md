# 整改子流程任务查询修复计划

## 问题描述

当前查询只指定了主流程的 `businessType="project_process:type:1"`，没有查询整改子流程 `declare_rectification` 对应的任务。

### 业务流程分析

```
主流程 (construction_process):
  发起人(144) → 省局审批(143) → [发起整改] → 审批人(1)
                                    ↓
整改子流程 (declare_rectification):     ← 当前查询遗漏！
  发起人(143) → 医院整改(144) → ???
                              ↑
                          用户144应该能看到这个任务
```

### 关联关系

整改子流程启动时，通过 `PARENT_PROCESS_INSTANCE_ID` 变量关联到主流程：
```java
childVariables.put("PARENT_PROCESS_INSTANCE_ID", event.getProcessInstanceId());
```

---

## 修改方案

采用 **主-子流程关联查询** 方式，确保只查询真正关联的整改子流程。

#### 2.4 新增私有方法（在文件末尾添加）

```java
/**
 * 查询主流程的整改子流程实例
 * 通过 PARENT_PROCESS_INSTANCE_ID 变量关联
 *
 * @param parentProcessInstanceIds 主流程实例ID列表
 * @return 整改子流程实例列表
 */
private List<org.flowable.engine.runtime.ProcessInstance> queryRectificationChildProcesses(
        List<String> parentProcessInstanceIds) {
    if (CollUtil.isEmpty(parentProcessInstanceIds)) {
        return Collections.emptyList();
    }

    try {
        // 查询所有包含 PARENT_PROCESS_INSTANCE_ID 变量的运行中流程实例
        List<org.flowable.engine.runtime.ProcessInstance> childProcesses = runtimeService
                .createProcessInstanceQuery()
                .variableValueEquals(PARENT_PROCESS_INSTANCE_ID, parentProcessInstanceIds.get(0))
                .list();

        // 过滤出真正属于这些父流程的子流程
        List<String> parentIds = new ArrayList<>(parentProcessInstanceIds);
        List<org.flowable.engine.runtime.ProcessInstance> matchedChildren = new ArrayList<>();

        for (org.flowable.engine.runtime.ProcessInstance child : childProcesses) {
            Object parentId = runtimeService.getVariable(child.getId(), PARENT_PROCESS_INSTANCE_ID);
            if (parentId != null && parentIds.contains(parentId.toString())) {
                matchedChildren.add(child);
                log.debug("[queryRectificationChildProcesses] 匹配到整改子流程: childId={}, parentId={}",
                        child.getId(), parentId);
            }
        }

        return matchedChildren;
    } catch (Exception e) {
        log.warn("[queryRectificationChildProcesses] 查询整改子流程失败: parentProcessInstanceIds={}", parentProcessInstanceIds, e);
        return Collections.emptyList();
    }
}
```

## 影响范围分析

### 影响的调用场景

| 调用位置 | 业务类型 | 影响 |
|---------|---------|------|
| `ProjectProcessList.vue` | `project_process:type:1` | ✅ 能查询到整改子流程任务 |
| `FilingList.vue` | `filing` | ✅ 能查询到整改子流程任务（如果有） |
| 其他业务列表 | 动态 | ✅ 能查询到整改子流程任务（如果有） |

### 不受影响的场景

1. **主流程任务查询**：原逻辑完整保留，只在后面补充整改子流程查询
2. **其他业务逻辑**：`getTaskBatchStatus` 等方法不受影响
3. **前端展示**：前端可以根据 `hasRectificationTodoTask` 字段决定是否显示整改任务

### 潜在风险

1. **性能影响**：查询整改子流程会增加一次数据库查询，但对于正常的业务量影响可忽略
2. **兼容性问题**：新增字段不会影响已有的返回结构，未使用的字段返回 `null`
